# Scenario Patterns

Ready-to-use patterns for common 3D web development scenarios. Each scenario includes tech stack recommendations, starter code, and production-ready patterns.

## Available Scenarios

### [Product Viewer](product-viewer.md)

Interactive 3D product showcase for e-commerce, portfolios, or marketing sites.

- OrbitControls with constrained rotation
- Studio lighting with HDR environment
- Color/material variants
- Hotspot annotations
- Screenshot export

**Best for**: Product pages, portfolio showcases, marketing sites

### [Data Visualization](data-visualization.md)

3D data visualization for dashboards, analytics, or interactive exploration.

- 3D bar charts, scatter plots, network graphs
- Spring-based animated transitions
- Zustand state binding for reactive updates
- Instanced rendering for large datasets

**Best for**: Analytics dashboards, data exploration, presentations

### [Simple Game](simple-game.md)

Browser-based 3D games with physics and interaction.

- Physics with @react-three/rapier
- Keyboard/pointer controls
- Game state management with Zustand
- Collectibles, hazards, moving platforms
- Audio integration with Howler.js

**Best for**: Casual games, interactive experiences, game jams

### [Immersive Landing](immersive-landing.md)

Scroll-driven 3D experiences for marketing or storytelling.

- ScrollControls with HTML overlay
- Parallax effects and camera paths
- Section-based animations
- Responsive viewport handling

**Best for**: Marketing pages, portfolios, storytelling sites

### [Configurator](configurator.md)

Interactive product configuration with real-time 3D preview.

- Dynamic material/color switching
- Accessory toggling
- Price calculation
- URL-based state sharing
- Screenshot/share functionality

**Best for**: E-commerce customization, product personalization

## Usage

Each scenario file includes:

1. **Tech Stack** - Recommended libraries and components
2. **Starter Code** - Copy-paste foundation
3. **Common Features** - Additional functionality patterns
4. **Performance Tips** - Optimization specific to the use case
5. **Checklist** - Pre-launch verification items

Reference these scenarios when starting a new project or when a user describes one of these use cases.
